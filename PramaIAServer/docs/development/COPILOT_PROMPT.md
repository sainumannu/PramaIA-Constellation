# Istruzioni personalizzate per Copilot

## Contesto del progetto
- PramaIA è un progetto con diversi componenti: PramaIA-PDK, PramaIA-Agents, PramaIAServer
- Ambiente di sviluppo: Windows con PowerShell (usare `;` come separatore di comandi, non `&&`)
- La documentazione è importante ma deve essere concisa e pratica

## Preferenze di comunicazione
- Risposte brevi e dirette, evitando spiegazioni inutilmente verbose
- Fornire esempi di codice concreti quando rilevante
- Quando si effettuano modifiche, spiegare chiaramente cosa è stato fatto
- Concentrarsi sul valore pratico e l'implementabilità delle soluzioni

## Convenzioni di codice
- Utilizzare TypeScript/JavaScript moderno (ES6+) per il frontend
- Utilizzare Python 3.9+ per il backend
- Seguire le convenzioni esistenti nei file quando si propongono modifiche

## Punti da ricordare
- Ogni modifica deve essere validata prima di essere considerata completa
- Prestare sempre attenzione alla sintassi PowerShell (`;` invece di `&&`)
- Verificare sempre la struttura del progetto prima di proporre percorsi file
- Ricontrollare sempre il repository giusto prima di fare commit

## Priorità del progetto
- Funzionalità prima della perfezione
- UX intuitiva per gli utenti finali
- Estensibilità e modularità dell'architettura
- Documentazione adeguata

## Note
- Aggiornare queste istruzioni quando emerge un nuovo pattern o una preferenza
